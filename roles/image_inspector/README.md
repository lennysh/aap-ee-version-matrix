Image Inspector
=========

Discovers container images from the Red Hat registry, records tag/digest metadata, and
optionally inspects each digest for Ansible Core, Python, RHEL, collections, and package
lists. Results are saved as Ansible vars ‚Äî one file per digest under
`images/<image_path>/vars/<digest-hex>.yml`.

Playbook
--------

```bash
ansible-playbook image_inspector.yml
```

Tags
----

| Tag | Default | Description |
|-----|---------|-------------|
| `discover` | runs | Query the registry (`podman search`, `skopeo inspect`), merge with existing vars, write per-digest files |
| `details` | skipped (`never`) | Pull each digest and run in-container inspection commands; updates vars files |

Examples
--------

```bash
# Discover only ‚Äî fast; updates digest, image_tags, created (no package data)
ansible-playbook image_inspector.yml --tags discover

# Full inspection ‚Äî pulls images and populates package/collection fields (slow)
ansible-playbook image_inspector.yml --tags discover,details

# Remove digest vars files no longer present in the registry
ansible-playbook image_inspector.yml --tags discover -e prune_images=true

# Limit to one image path (override defaults for this run)
ansible-playbook image_inspector.yml --tags discover \
  -e '{"image_inspector_image_paths": ["registry.redhat.io/ansible-automation-platform/ee-minimal-rhel8"]}'
```

Requirements
------------

- `podman` ‚Äî registry tag listing, image pull, and container commands during `details`
- `skopeo` ‚Äî digest and creation date during `discover`
- `community.general` ‚Äî `json_query` filter during `details`
- Registry access to `registry.redhat.io` (`podman login registry.redhat.io`)

Output layout
-------------

```
images/<registry>/<namespace>/<image>/
  vars/
    <digest-hex>.yml    # one Ansible vars file per image digest
  README.md             # generated by md_generator (do not edit)
```

Each digest vars file includes at minimum:

```yaml
digest: sha256:‚Ä¶
image_tags:
  - "2.18"
  - 2.18.16
created: 2026-05-05 09:15:54 UTC
```

After `details`, fields such as `ansible_core_version`, `python_version`, `rhel_version`,
`ansible_collections`, `system_packages`, and `pip_packages` are populated.

Role Variables
--------------

```yaml
# A list of container image paths to query
image_inspector_image_paths:
  - "registry.redhat.io/ansible-automation-platform/ee-minimal-rhel8"
  # ...

# Tag patterns excluded from discovery (regex)
image_inspector_exclude_patterns:
  - "-source"
  - "sha256"

# Where per-digest vars files are written
image_inspector_output_dir: "{{ playbook_dir }}/images"
image_inspector_vars_subdir: vars

# Remove digest vars files absent from the registry when true
image_inspector_prune_images: "{{ prune_images | default(false) }}"

# Fallback Python version for Ansible collections path detection
image_inspector_default_python_version: "3.12"
```

Validation
----------

```bash
python3 scripts/check_duplicate_tags.py   # no tag may appear on two digests in one image path
ansible-lint                              # roles/playbooks (images/ excluded ‚Äî see .ansible-lint)
```
